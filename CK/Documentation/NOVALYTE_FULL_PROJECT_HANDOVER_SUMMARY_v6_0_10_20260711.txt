V6.0.9 CORRECTIVE CHANGES
1. Recommended category default
- Recommended remains the first category option.
- Recommended is forcibly selected on every fresh client-site load, even when an older browser state saved All categories.
- Platform filtering continues to combine with Recommended; for example, Facebook + Recommended shows only recommended Facebook services.

2. Admin Recommend control
- The compact Recommend column and checkbox/toggle remain in Admin Services.
- Checking or unchecking saves immediately without refreshing the admin page.
- The Recommended setting remains part of the client-safe public service data.

3. Client order board service ID
- Client order rows display the selected provider/service ID, such as 1723 or 1097, instead of generated internal order keys.
- The public order payload now preserves providerId.
- The client view can resolve older records by matching the internal service record or service name when providerId is missing.
- Search continues to support service ID, client name, and service name.
- The Supabase novalyte_list_orders RPC now returns provider_id, item_id, and service_id.

4. Orders shown arrangement
- The Orders shown label is placed above the live number.
- Label and number are centered as one compact vertical counter.

5. Admin Order History repair
- Added a deterministic colgroup for all ten Order History columns.
- Desktop Order History is fixed to the available width and does not use a horizontal scrollbar.
- The table wrapper is no longer horizontally scrollable on desktop.
- Date, Client, Service, Qty, Provider, Client Fee, Revenue, Status, Payment, and Actions stay aligned.
- View, Edit, and Void/Undo Void use one compact row and stay inside the Actions column.
- Status controls and payment badges remain compact and readable.
- The table scroll position is reset to the left whenever orders are rendered, preventing stale horizontal offsets from older builds.

6. Client layout refinement
- Client home/order tracker uses a narrower maximum width and reduced card/control spacing.
- Live count cards and order rows are compact while preserving readability.
- The client order table uses explicit stable column widths.
